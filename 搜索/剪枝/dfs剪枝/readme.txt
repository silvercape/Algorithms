# FDCS数据结构20201026上机题目

## 题目背景

光华楼的电梯被入侵了，现在它每层楼只有上或下两个功能，请帮帮大家，计算从某一层到另一层至少要按几次按钮。

## 题目描述

光华楼一共$N$层，第$i$层能上或者下的楼层是固定的$K_i$层，如果上或者下之后的楼层超过$N$或者小于$1$，那么这次按键将不生效。例如，在一个三层的楼里，给出的$K$数组为$2,1,1$，则意味着在一楼只能按上键直接抵达三楼，在二楼可以按下键抵达1楼或者按上键抵达三楼，在三楼只能按下键抵达二楼。

## 输入格式

输入共有两行

第一行为三个数$N,S,E$，分别表示总楼层数，起始楼层和目标楼层

第二行有N个数，第$i$个数$K_i$表示当前楼层按键能上或者下的层数。

## 输出格式

输出为一个数，为从A层到B层所需的最小按键次数

## 样例 #1

### 样例输入 #1

```
5 1 5
2 3 1 3 5
```

### 样例输出 #1

```
3
```

## 提示

在一楼按上键抵达三楼，然后按下键抵达2楼，最后再按上键即可抵达五楼

输入数据保证$ 0 \lt N \le 200$， $0 \lt S,E \le N $
